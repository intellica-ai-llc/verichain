AGENT‑SEED v15.2 — Autonomous Systems Language (ASL) User Manual

Edition Notice This manual covers ASL v15.2 (Edition 2029), the definitive version of the Autonomous Systems Language. The reference implementation is the Agent Seed toolchain (seedc, seedvm, seedpkg, seedls, seedfmt, seeddbg). Foreword: The Philosophy of ASL The Autonomous Systems Language (ASL) is a programming language purpose‑built for autonomous agentic systems. It is not a general‑purpose language adapted to agents; it is a language whose every construct—from the type system to the runtime—is designed to express, constrain, and execute autonomous behaviour safely, deterministically, and with full audibility. ASL is grounded in the following fundamental principles: Conformance is verified against the ASL‑CONF‑15 test suite. 1. Composition that Cannot Multiply Errors — Computation and the Uncertain Monad The O'Reilly analysis frames multi-agent systems as "probabilistic pipelines, where every unvalidated handoff multiplies uncertainty." Liu's λ_A calculus provides the formal foundation. AGENT‑SEED provides the concrete implementation. In ASL, no value exists outside a Computation. Every computation carries an uncertainty interval, taint influence, cost bounds, capability requirements, and provenance reference. The discharge expression — the only way to unwrap a computation — must explicitly check all thresholds: text discharge findings with { confidence: 0.85, taint: 0.2, budget: remaining } { synthesize(findings) } The U1–U6 axioms of Uncertain provide a formal probability monad with interval semantics. U2 (Bind) multiplies intervals through composition, giving the programmer a direct, type-checked view of how uncertainty compounds: text // Uncertainty accumulates at every pipeline stage — compiler tracks it let result = infer(...) // [0.88, 0.94] |> infer(...) // [0.85, 0.92] |> infer(...); // [0.90, 0.95] // result: Uncertain[0.673, 0.822] — computed automatically (... continue existing intro content ...)

Part I – Getting Started

1. Installation

The Agent Seed toolchain is under active development and currently built from source. Pre‑built binaries and package registries are planned for the first public release. 1.1 Platform Support & Prerequisites ASL runs on Linux (x86_64, arm64), macOS (x86_64, arm64), and Windows (x86_64). The only prerequisite is a recent Rust toolchain (stable, ≥1.85). Install Rust if you don't have it: bash curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh source "$HOME/.cargo/env" 1.2 Building from Source Clone the repository and build the workspace: bash git clone https://github.com/agentseedlanguage-cpu/agentseed.git cd agentseed cargo build --workspace --release The binaries are placed in target/release/: seedc – the compiler seedvm – the virtual machine seedpkg – the package manager (scaffold) seedls – the language server (scaffold) seedfmt – the code formatter seeddbg – the debug adapter Add target/release/ to your PATH for convenience, or run the binaries directly. 1.3 Install via npm / curl (coming soon) bash npm install -g @agentseed/cli # not yet published curl … | sh # not yet available These distribution channels will be available with the first public release. 1.4 Verify Installation bash ./target/release/seedc --version # seedc 0.1.0 ./target/release/seedvm --version # seedvm 0.1.0

2. Hello, Agent

A minimal ASL program is an agent with a main function. Create a file named hello.seed: seed agent hello { fn main() -> i32 { print("Hello, Agent!"); 0 } } Compile it to bytecode: bash ./target/release/seedc build hello.seed -o hello.aslb Run the compiled module on the VM: bash ./target/release/seedvm run hello.aslb Output: text Hello, Agent! The seedc build command compiles the source to a binary .aslb module. The seedvm run command executes that module deterministically.
